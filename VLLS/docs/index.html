VisualLangLab is a visual parser-generator IDE for developing parsers without any textual grammar specification (BNF/EBNF/PEG), script or code. It is simple in use, and easy to learn. Unlike other parser-generators, VisualLangLab represents parser rules as visual grammar-trees with distinct icons for the grammar-tree nodes as in the figures below. These example grammars are included with the VisualLangLab download.

Examples from Programming in Scala, 2e (book) Examples from The Definitive ANTLR Reference (book)

The grammar-trees are equivalent to PEGs, and can handle LL(*) grammars with left recursion. They are also executable, and can be run at any time at the click of a button. No additional code, skills, or tools are needed. Input for the parser can be entered into the IDE or obtained from user-nominated files. This simplifies testing, and promotes an iterative-incremental development process that accelerates development. Optional semantic actions written as Javascript or Scala functions can also be associated with the grammar-tree's nodes.

Parser Development and Use

The parser development process is reduced to creating the equivalent grammar-tree(s) using menu-bar and context-menu operations of the GUI. Textual input from the keyboard is required only for naming the parser rules and tokens, and for specifying regular expression patterns for tokens. The grammar is saved as a XML file that can be reloaded for review, further editing or testing.

An API enables client programs in any JVM language to load the XML file and regenerate the parser. Scala's parser combinators are used to turn tree (or XML) data directly into a Parser at run-time without code-generation, so parser regeneration is very fast.

VisualLangLab is written in Scala, and leverages its parser combinators to produce backtracking, recursive-descent parsers. But the user does not have to know Scala, and a Scala installation is not required for running it. VisualLangLab's dependence on Scala is described here.

Download, Installation, and Running

You only need the all-inclusive jar file VLLS-All.jar to run VisualLangLab. Double-clicking this file starts the VisualLangLab GUI. This jar file includes all libraries required, so you do not have to install Scala or any other software. It also includes the Sample Grammars, but not the HTML documentation (which are in the VLLS.zip file described below.

The latest-release file (VLLS.zip) contains all the VisualLangLab class files and the HTML documentation, but not the Scala libraries. You can use this file to run VisualLangLab only if you have a Scala installation.

Prerequisites

If you are using VLLS-All.jar the only prerequisite is a 6.0+ JRE. You can create, test, and run parsers within the IDE without having to install Scala—not even if the grammars contain Scala functions in their action-code

Scala 2.9.0+ is required only for separately developing and/or running application programs that use the API. You can download Scala from here.

History

VisualLangLab was born on 5th March 2004 as A Visual LL(K) Parser Generator. This first version was written in Java, and can still be found here. The software was usable by mid-2004, and an interesting application is described in this java.net article published in early October 2004.

The availability of Scala and its built-in parser-combinators prompted a re-write, and a Scala version was published under this java.net project on 10th March 2010. Although all of the code was re-written (in Scala), the essential character of the GUI remains the same.
