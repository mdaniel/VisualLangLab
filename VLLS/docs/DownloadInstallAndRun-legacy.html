VisualLangLab - Download, Install, and Run

There are two files available for download. To get started quickly just download the all-inclusive jar file (immediately below here).

All-Inclusive JAR File (Quick-Start)

For all practical purposes, you only need VLLS-All.jar to run VisualLangLab. Double-clicking this file starts the VisualLangLab GUI. The only pre-requisite is a 6.0+ JRE. Users with Unix-like systems will first need to perform a "chmod +x VLLS-All.jar".

This jar file includes all libraries required, so you do not have to install Scala or any other software. It includes the Sample Grammars, but not the HTML documentation (which are in the VLLS.zip file described below). It also includes the readme.txt, copying.txt, changes.txt, and licenses files described below.

  • readme.txt - Summary information about VisualLangLab
  • copying.txt - Copyright information
  • changes.txt - Changes from previous version
  • licenses - Directory containing copyright files

Latest Binary-Release ZIP File

The latest-release file (VLLS.zip) contains all the VisualLangLab class files and the HTML documentation, but not the Scala libraries. You can use this file to run VisualLangLab only if you have a Scala installation. This zip file contains the following files/directories:

  • VLLS.jar - the run-time class-library
  • vlls.bat - a launcher for Windows. It assumes that VLLS.jar is also in the same directory
  • vlls - a launcher for UNIX-like environments. It assumes that VLLS.jar is also in the same directory
  • readme.txt - Summary information about VisualLangLab
  • copying.txt - Copyright information
  • changes.txt - Changes from previous version
  • index.html - Documentation in HTML
  • docs - Directory containing documentation files
  • licenses - Directory containing copyright files

There is no particular installation action required. After unzipping the fle to a directory, VisualLangLab can be started by invoking one of the launcher scripts -- vlls.bat for Windows, or vlls for UNIX-like systems. Both launchers assume that VLLS.jar is also in the same directory.
