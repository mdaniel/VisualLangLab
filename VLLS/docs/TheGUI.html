The VisualLangLab GUI

Figure-1 below depicts the complete VisualLangLab GUI. The following description details all its features.


Figure-1. The VisualLangLab GUI

The Menu Bar

The Menu Bar (across the top of the GUI) supports the following functions:

  • File - New, Open, Save, SaveAs, and Exit operations
  • View - enable/disable display of additional grammar-tree details, and choose Java look and feel
  • Tokens - New literal, New regex, Edit, Find, Delete, Import, and Export operations
  • Parsers - New, Rename, Find, and Delete operations
  • Globals - define the regular-expression for whitespace and comments
  • Test - runs parser with input from Parser Test Input area or from user-specified file. Also enables choice of pretty-printers for AST, and enabling Trace all (parsers)
  • Log - copy or clear Parser Log area
  • Help - documentation, sample grammars, and the About page

The Tool Bar

The Tool Bar just below the Menu Bar enables quick invocation of commonly used operations:

  • File - New, Open, Save, and SaveAs operations
  • Back button to display last viewed grammar-tree
  • Combo box to select grammar-tree by name
  • Parsers - New, Rename, and Find operations
  • Tokens - New literal, New regex, Edit, Find, Import, and Export operations
  • Test - run and stop parser operations
  • Log - Copy, and Clear operations

The Grammar Tree

The Grammar Tree is the JTree at the top left of the GUI (below the toolbar). A context menu displayed when any node of the grammar-tree is right-clicked (as in Figure-1 above) supports all the grammar editing functions as described here.

The Parse Tree (AST) Area

The Parse Tree (AST) Area is the TextArea in the middle (below the toolbar). VisualLangLab displayes the structure of the AST produced by the selected grammar-tree node here.

The format of the displayed information is described here.

The Action-Code Area

The Action-Code area is the TextArea on the right (below the toolbar). This editable area can be used to optionally provide any action-code for the selected grammar-tree node.

The action-code must either be a Javascript function that takes one argument, or a Scala function taking one argument of type Any. More details and examples can be found here.

The Testing Area

The two TextAreas along the bottom of the GUI (Parser Test Input and Parser Log) constitute the testing area.

The editable Parser Test Input area on the left is used for providing test input to the parser under development. Any text typed into this area is read by the parser under development when the Parse input toolbar button is clicked (or the Test -> Parse input menu item is selected.

The Parser Log area is where any output from the parser run appears. Error messages are printed in red, while normal output is printed in black. When a parser runs to completion successfully, the AST produced is printed out along with some run-time and memory utilization statistics.
