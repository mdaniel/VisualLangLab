How is "Choice" related to Scala types?

The "Choice" node type is transformed into multiple invocations of the "|" combinator. In effect, all the child nodes of a Sequence are combined together with applications of "|". Thus, a Choice node that has x, y, and z as child nodes effectively represents the parser "(x | y | z)". The name "Choice" has no relationship to any classes in the Scala API.

Where is the generated code?

While you can use VisualLangLab to visually model a parser and also test-run it, the generated code option has not yet been completed. VisualLangLab does not need to generate or compile any code for its own purposes -- which is how it is able to run the created parser. "Creating a parser" in the VisualLangLab context really means just visually modeling it!

How are these concepts related to Scala combinators?

Though not the whole truth, it is useful to believe that the GUI is derived from "scala.util.parsing.combinator.Parsers", and that the trees are related to "Parsers.Parser"

How do I edit a grammar-node?

You can edit a grammar-node by right-clicking the node and choosing "Edit" from the context menu. The sub-menu under "Edit" allows you to move the node up or down (relative to its siblings), and also to cut, copy, paste, and delete the selected node.

How do I assign a custom error message to a grammar-node?

You assign a (custom) error-message to a grammar-tree node by right-clicking it and choosing "Error message" from the context menu. Then enter the message text into the dialog box presented.

What is "flattening"?

You instruct VisualLangLab to flatten all nested "~" objects by choosing "Run -> Flatten nested ~" from the main menu. When flattening has been enabled in this way, VisualLangLab translates each nested "~" object (in the parse-tree) into a Scala Array. This makes life easier for the parse-tree processor.

How do I create a Literal token?

You create a Literal token by choosing Tokens -> New literal from the main menu. Then enter literal-name, a comma, and the literal-value into the dialog box presented. For example, to create a Literal called "DASH" for the character "-", enter "DASH,-" (without the quotes) into the dialog. You can add blanks around the comma if you wish.

How do I set the multiplicity of a grammar node?

You set the multiplicity of a node in the grammar-tree by right-clicking the node and choosing Multiplicity -> ? from the context menu. The "?" in the preceding sentence represents any of the options availabile (1, *, +, ?, and 0).

Why/when would I set multiplicty to 0 (zero)?

Setting Multiplicity to 0 means that matching data must not exist in the input. Check documentation about the "not" combinator.

How do I create a parser?

You create a new parser by choosing "Parsers -> New parser" from the main menu, and entering the parser name into the dialog box presented. This creates (and displays) a newly created parser with just a root node. You can then proceed to build the complete parser by right-clicking the root-node, and choosing "Add" from the context menu. For the simplest possible parser you would add a token (a previously created Literal or a Regex) directly to the root-node. But more typically you would add one of the aggregate types (Sequence, Choice, or RepSep) to the root-node. You would then recursively add further elements to that grammar-tree till the parser is fully defined.

How do I create a Regex token

You create a Regex token by choosing Tokens -> New regex from the main menu. Then enter regex-name, a comma, and the regex-pattern into the dialog box presented. For example, to create a Regex called "INT" for the pattern "\d+", enter "INT,\d+" (without the quotes) into the dialog. You can add blanks around the comma if you wish.

Why is there no Rep1Sep?

The "RepSep" node can perform the function of both combinators "repsep" and "rep1sep". Which one it emulates depends on the multiplicity setting. A multiplicity setting of "*" gives the "repsep" behavior, while a setting of "+" gives the "rep1sep" behavior.

How do I run VisualLangLab?

You run VisualLangLab by executing the following command line: 
scala -cp VisualLangLab.jar net.java.dev.vll.VisualLangLab
You must have Scala 2.8 installed. VisualLangLab has been tested on Windows-XP and Linux (Ubuntu 9.10)

How do I run the parser?

You run the created parser by pasting the input text into the "Parser input" area, and then choosing "Run -> Run parser" from the main menu. Remember to have the top-level parser selected in the tree-display when you invoke "Run -> Run parser". Running the parser without any text in the "Parser input" area passes an empty string ("") to the parser.

How is "Sequence" related to other Scala types?

The "Sequence" node type is transformed into multiple invocations of the "~" combinator. In effect, all the child nodes of a Sequence are threaded together with applications of "~". Thus, a Sequence that has a, b, and c as child nodes effectively represents the parser "(a ~ b ~ c)". The name "Sequence" has no relationship to similarly named classes in the Scala API.

How can I get a run-time trace of a grammar-node?

You can enable (or disable) tracing of a grammar-node by right-clicking the node and choosing "Trace" from the context menu. When tracing is enabled on a node, VisualLangLab prints a trace message whenever the trace-enabled node is used (successfully or otherwise) in the course of a parser run.

How do I troubleshoot a parser?

There are two features that help in troubleshooting a parser -- tracing and custom error messages.
